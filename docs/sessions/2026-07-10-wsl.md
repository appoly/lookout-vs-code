## Decisions made

- Optimize around native VS Code surfaces; custom webviews are not the product foundation.
- Treat explicit hooks/events as authoritative and terminal scraping as out of scope.
- Keep usage unavailable rather than estimated.
- Describe Git results as workspace changes from a captured commit, not agent authorship.
- Treat process lifetime as active/unknown work state and use provider events for working/waiting state.
- Classify configured plans/docs as review artifacts rather than ordinary code changes.
- Keep review/usage available in untrusted workspaces but gate command execution.
- Do not persist custom commands.

See [the durable decision log](../DECISIONS.md) and [roadmap](../ROADMAP.md).
